OPTION 1: OLLAMA (BEST FOR OFFLINE - RECOMMENDED FOR FIRST TIME!)
================================================================================

This is the easiest - runs 100% locally on your laptop. No API keys needed!

STEP 1: Download and Install Ollama
   📥 Go to: https://ollama.com/download
   🖱️  Download for Windows
   💻 Run the installer
   ✅ Verify: Open Command Prompt and type:
      ollama --version

STEP 2: Pull an AI Model (Choose ONE)

   Option A: LIGHTWEIGHT (Recommended for most laptops)
   👉 Run in Command Prompt:
      ollama pull llama3.2:3b

   Specs: ~2 GB download, needs 4 GB RAM, very fast (1-2s responses)

   Option B: SMARTER (If you have 8GB+ RAM)
   👉 ollama pull llama3.1:8b

   Specs: ~5 GB download, needs 8 GB RAM, slower but smarter

   Option C: BALANCED
   👉 ollama pull phi3:mini

   Specs: ~2.5 GB download, needs 4 GB RAM

STEP 3: Test Ollama is Running
   Run this command:
      ollama serve

   You should see: "Listening on 127.0.0.1:11434"

   Keep this terminal window open. Ollama runs in background after this.

STEP 4: Verify JARVIS Auto-Detects Ollama
   Run JARVIS:
      python main.py

   You should see in logs:
      ✅ Local LLM ready: llama3.2:3b — no API needed! ✅

DONE! JARVIS now uses offline AI. No API key needed, unlimited usage!

OPTION 2: GROQ API (FASTEST FREE - 30 REQUESTS/MINUTE)
================================================================================

Groq is INCREDIBLY fast (0.5-1 second responses). Free tier includes 30 req/min.

STEP 1: Sign Up for Groq
   🌐 Go to: https://console.groq.com
   📧 Sign up with email (free account)
   🔑 Go to API Keys section
   ➕ Click "Create API Key"
   📋 Copy the key (starts with "gsk_")

STEP 2: Add to JARVIS
   1. Open .env file in JARVIS folder
   2. Add this line:
      GROQ_API_KEY=gsk_YOUR_KEY_HERE
   3. Save and restart JARVIS

STEP 3: Test Groq
   JARVIS will automatically try Groq when local LLM is busy or unavailable.

   In logs you'll see:
      ✅ Groq API available (30 req/min)

FEATURES:
   ✅ 30 requests/minute (Free tier)
   ✅ Ultra-fast (0.5-1 second)
   ✅ Llama 3.2 models available
   ✅ No quota resets needed

================================================================================
OPTION 3: HUGGINGFACE API (UNLIMITED - RATE LIMITED)
================================================================================

HuggingFace gives you unlimited requests with gentle rate limiting.

STEP 1: Create HuggingFace Account
   🌐 Go to: https://huggingface.co/join
   📧 Sign up (free account)

STEP 2: Get Your API Token
   1. Go to: https://huggingface.co/settings/tokens
   2. Click "New token"
   3. Name it "JARVIS"
   4. Set type to "Read"
   5. Click "Create token"
   6. Copy the token (starts with "hf_")

STEP 3: Add to JARVIS
   1. Open .env file
   2. Add this line:
      HUGGINGFACE_TOKEN=hf_YOUR_TOKEN_HERE
   3. Save and restart JARVIS

FEATURES:
   ✅ Unlimited requests (gentle rate limiting)
   ✅ 1000+ models available
   ✅ No quota resets
   ✅ Works reliably

================================================================================
TROUBLESHOOTING
================================================================================

Problem: "Ollama not running"
   Solution:
   1. Make sure Ollama is installed
   2. Open terminal and run: ollama serve
   3. Keep that terminal window open while using JARVIS

Problem: "GROQ_API_KEY error"
   Solution:
   1. Check your .env file has correct key format (gsk_...)
   2. Restart JARVIS after adding key
   3. Check logs for "✅ Groq API available"

Problem: "HuggingFace returning empty"
   Solution:
   1. Verify token is correct (hf_...)
   2. Check token has "Read" permission at huggingface.co/settings/tokens
   3. Wait 30+ seconds between requests (HF rate limiting)

Problem: "All APIs failing"
   Solution:
   1. Check internet connection
   2. JARVIS will fall back to system commands (still works!)
   3. Most features still available without AI
