WANT UNLIMITED RESPONSES? Add HuggingFace:
   1. Sign up: https://huggingface.co (free account)
   2. Get token: https://huggingface.co/settings/tokens
   3. Open .env file
   4. Add: HUGGINGFACE_TOKEN=hf_YOUR_TOKEN_HERE
   5. Save and restart JARVIS
   ✅ Done! Now you have truly unlimited responses!

WANT SMART RESPONSES? Add Google AI Studio:
   1. Go to: https://aistudio.google.com/app/apikey
   2. Click "Create API Key"
   3. Open .env file
   4. Add: GOOGLE_AI_STUDIO_KEY=AIza_YOUR_KEY_HERE
   5. Save and restart JARVIS
   ✅ Done! Now you have very smart responses!

HOW IT WORKS
================================================================================

When you talk to JARVIS, it tries in this order:

1. Local Ollama (fastest, offline, unlimited)
   ↓ (if not available)
2. Cache (instant for repeated questions)
   ↓ (if not found)
3. Groq API (0.5 seconds)
   ↓ (if quota hit)
4. HuggingFace (2-3 seconds, unlimited)
   ↓ (if fails)
5. Google AI Studio (2-5 seconds, smart)
   ↓ (if all fail)
6. System commands still work (open apps, weather, news, etc)

Result: You ALWAYS get a response! Never run out of quota!

================================================================================
TROUBLESHOOTING
================================================================================

Problem: "Ollama not detected"
Solution:
   → Make sure ollama serve is running
   → Open new Command Prompt window
   → Type: ollama serve
   → Keep that window open while using JARVIS

Problem: "API key error"
Solution:
   → Check .env file for typos
   → Make sure no extra spaces or quotes
   → Check key format:
      GROQ_API_KEY starts with gsk_
      HUGGINGFACE_TOKEN starts with hf_
      GOOGLE_AI_STUDIO_KEY starts with AIza_
   → Restart JARVIS

Problem: "Getting slow responses"
Solution:
   → Use Groq (fastest: 0.5s)
   → Or use Ollama if offline
   → Check internet connection
   → System might be busy

Problem: "JARVIS not responding"
Solution:
   → Check if ollama serve is running
   → Check internet connection
   → Check logs: data/jarvis.log
   → Run: python verify_installation.py
   → Restart JARVIS
