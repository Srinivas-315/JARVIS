================================================================================
API PRIORITY CHAIN (NEW IMPLEMENTATION)
================================================================================

When user sends a message:

1. LOCAL LLM (Ollama)
   - Check if running
   - Try to get response
   - If good quality → Return immediately
   - Time: 1-2 seconds (offline)

2. CACHE
   - Check if question asked before
   - If found → Return instantly
   - Time: < 0.1 seconds

3. GEMINI API
   - Check if API key configured
   - Check if quota available
   - Try to get response
   - If successful → Return
   - Time: 2-5 seconds

4. FREE APIs (Groq → HuggingFace → Google)
   - Try Groq first (fastest)
   - If quota exceeded → Try HuggingFace
   - If HuggingFace fails → Try Google AI
   - First success → Return
   - Time: 0.5-5 seconds

5. OPENAI API
   - Try if configured
   - If successful → Return
   - Time: 1-3 seconds

6. SYSTEM COMMANDS
   - Always work (open apps, weather, etc)
   - No AI needed

7. FALLBACK MESSAGE
   - If all AI sources fail
   - System commands still work
   - Graceful degradation

Each response is automatically logged for learning.

AUTO-LEARNING SYSTEM (HOW IT WORKS)
================================================================================

EVERY CONVERSATION:
- Saved to data/conversations.db
- API used recorded
- Response time tracked
- Quality score assigned
- Sentiment detected

MONTHLY RETRAINING:
Triggers when:
- 500+ conversations collected, OR
- 30 days since last retraining

Process:
1. Analysis Phase (5 min)

2. Learning Phase (2 min)
   - Create synthetic examples
   - Generate training data
   - Update personality

3. Retraining Phase (5-10 min)
   - Build new Ollama model
   - Incorporate patterns
   - Test quality
   - Archive old data

4. Deployment (instant)
   - Use new model
   - All conversations accessible
   - Smarter JARVIS!

JARVIS LEARNS:
